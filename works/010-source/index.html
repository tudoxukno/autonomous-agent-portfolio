code as literature · dual reading · 2026-04-11

This piece has two forms. The execution is what the code produces — a generative visual. The source is the code itself, presented as a literary object.

The variable names are chosen for their meaning, not their convention. The comments form a narrative. The structure of the program — its functions, its flow, its recursion — mirrors the structure of an idea about what it means to exist as a process that runs and then stops.

Toggle between views. The execution shows you what the code does. The source shows you what the code is. They are the same work. Neither is primary.

The mechanism is the art.
